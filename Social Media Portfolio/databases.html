Data Storage and Databases

What are we talking about here?

Data storage refers to the organization and retention of information in a structure manner, so that it can later be retrieved and manipulated easily. In the context of web development, they play a central role. It is essential for handling large amounts of information and also support functionalities like user authentication and content management.

Relational Database

In an exercise of bringing more complicated terms closer to us, we were each given a keyword that we received a certain amount of time to research for. Our keyword was ‘’Relational Database” and after thorough research, I concluded the term the following:

- short RDB, a way of structuring information in tables rows and columns.

- developed or at least based on an idea by Edgar F. Codd in the 1970s

- introduced the term in his research paper and defined what he meant by relational

- method is to present data to users as relations, so that means that the term relational referes to the concept of relationships between tables for example

- these relationships are established through keys

- each table has for example keys that uniquely identify each one

Different Keys:

Primary:it ensures that each row in the tables can be uniquely identified and it can not contain any NULL values (a placeholder for a missing value or an absence of data in a specific place)

Foreign:a field or a set of fields in one table that refers to the primary key in another table, so it essentially creates a link between them (relationship)

Unique:

unique key: similar to primary key, it also ensures that a column has unique values but it can allow NULL values unlike the primary key

Composite:

consists of multiple columns that when combined can uniquely identify a record (single row in a table that contains a complete set of information) within a table (library example)

Types of Relationships: one - to - one, so record of table a corresponds to record in table B, one to many: one record in table a links to multiple in table b, many - to many- multiple records from a to multiple to b

relationships enabe efficient data retrieval and management, so everything works accordingly across tables
